Phần 1: Lý thuyết (40 điểm)

Câu 1: position: absolute và position: relative

Khác biệt: relative định vị phần tử tương đối so với vị trí bình thường của chính nó (không làm mất không gian cũ). absolute định vị phần tử tương đối so với phần tử cha gần nhất có thuộc tính position khác static (nó bị rút khỏi luồng tài liệu, không chiếm không gian).

Ví dụ Responsive: Khi tạo một tooltip (chú thích). Nút bấm chứa tooltip sẽ có position: relative, còn hộp tooltip bên trong sẽ có position: absolute. Dù xem trên thiết bị nào, tooltip luôn bám theo nút bấm đó.

Ảnh hưởng Deploy: CSS thuần túy không ảnh hưởng trực tiếp đến quá trình deploy server, nhưng responsive layout đảm bảo khi website được host và truy cập từ internet, giao diện sẽ không bị vỡ trên các thiết bị (điện thoại, máy tính) khác nhau.

Câu 2: Domain, Hosting và Deploy

Quy trình: Bạn không cấu hình hosting trực tiếp bằng JavaScript, mà code JS của bạn sẽ được tải lên Hosting. Bạn mua Domain (tên miền) -> Đăng ký Hosting -> Upload code (HTML/CSS/JS) lên Hosting -> Cấu hình DNS.

Vai trò DNS: Domain Name System đóng vai trò như danh bạ điện thoại, dịch tên miền (ví dụ: tenban.com) thành địa chỉ IP của máy chủ Hosting chứa mã nguồn của bạn.

Công cụ không cần server riêng: GitHub Pages, Netlify, Vercel.

Rủi ro bảo mật: Nếu đưa các API Key nhạy cảm (như key database) vào code JavaScript frontend, khi deploy công khai, người dùng có thể dễ dàng lấy được key này qua DevTools và đánh cắp dữ liệu.

Câu 3: let, const và var

So sánh: var có scope ở cấp hàm (function scope), có hoisting và khởi tạo là undefined. let và const có scope ở cấp khối (block scope), có hoisting nhưng rơi vào vùng "Temporal Dead Zone" (không thể truy cập trước khi khai báo). const không thể gán lại giá trị.

Ví dụ lỗi: console.log(a); let a = 5; (Lỗi ReferenceError do truy cập let trước khi khởi tạo).

Cách Debug: Nhấn F12 mở Chrome DevTools, qua tab "Sources", đặt Breakpoint tại dòng code nghi ngờ để theo dõi sự thay đổi của biến trong phần "Scope".

Câu 4: Event Delegation

Khái niệm: Là kỹ thuật gắn một Event Listener duy nhất vào phần tử cha, thay vì gắn nhiều Listener vào từng phần tử con. Nó tận dụng cơ chế Event Bubbling của DOM.

Ví dụ: Một danh sách <ul> động có thể thêm/bớt thẻ <li>. Thay vì gắn sự kiện click cho từng thẻ li, ta gắn cho ul. ul.addEventListener('click', (e) => { if(e.target.tagName === 'LI') console.log('Đã click LI'); })

Lợi ích hiệu suất: Tiết kiệm bộ nhớ (chỉ có 1 listener thay vì hàng trăm). HTML/CSS render nhanh hơn và không cần phải gán lại sự kiện bằng JS mỗi khi tạo thêm một phần tử DOM mới.
